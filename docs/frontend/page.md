# MeetingPilot 前端页面清单文档

## 1. 仪表盘 / 会议列表页 (Dashboard)

**功能定位**：系统的入口，用于管理历史会议和快速发起新任务。

* **核心功能**：
* **会议概览卡片**：展示会议标题、日期、时长、AI 摘要简述。
* **快速操作**：支持一键上传音频文件或开始录音。
* **状态追踪**：显示会议当前的 AI 处理状态（转录中、分析中、已完成、失败）。


* **UI 设计要点**：
* 采用 **Grid（网格）布局**，配合圆角卡片。
* 使用卡通猫 Coach 的占位图或空状态引导，保持简约可爱风格。


* **后端对接**：`GET /meetings`。

## 2. 会议详情工作台 (Meeting Workspace) —— **核心页**

**功能定位**：用户停留时间最长的页面，实现 PRD 亮点的"可追溯性"交互。

* **核心组件布局**：
* **顶部/悬浮区**：音频波形播放器（Waveform.vue），实时显示播放进度。
* **左侧主区**：转录对话流（TranscriptList.vue）。
* **交互**：点击文字跳转音频，音频播放自动高亮对应文字。


* **右侧分析区**：多页签面板。
* **Tab 1 - 智能纪要**：展示 Summary Agent 提取的结构化信息。
* **Tab 2 - 待办追踪**：展示 Action Agent 提取的任务列表，支持勾选完成。
* **Tab 3 - 智能问答**：与 QA Agent 进行基于 WebSocket 的流式对话。




* **UI 设计要点**：
* **双栏或三栏布局**，确保用户可以边看原文边看 AI 摘要。
* **高亮交互**：当前播放行使用浅紫色背景，点击行增加轻微的位移动画。


* **后端对接**：`GET /meetings/{id}`、`GET /transcripts`、`GET /summary`、`WS /chat`。

## 3. 会议上传与处理页 (Upload & Processing)

**功能定位**：缓解用户在长音频处理过程中的焦虑感。

* **核心功能**：
* **文件上传器**：支持拖拽上传，校验文件格式。
* **实时状态反馈**：展示"正在上传" -> "正在转录" -> "正在由 Agent 分析"的进度条。
* **Agent 状态机视觉化**：模仿 LangGraph 的节点流向，用简单的动画展示当前是哪个 Agent 正在工作。


* **UI 设计要点**：
* 居中对齐，使用大量的微交互动画。


* **后端对接**：`POST /meetings/upload`。

## 4. 待办任务总览页 (Task Overview)

**功能定位**：跨会议维度统一管理所有待办事项，弥补会议详情页仅展示单会议待办的不足。

* **核心功能**：
* **全局任务列表**：聚合所有会议的 ActionItem，支持按状态（todo / doing / done / canceled）、优先级（low / medium / high / urgent）、来源会议筛选。
* **任务操作**：支持勾选完成、编辑内容、删除、手动创建新任务。
* **来源追溯**：点击任务可跳转到对应会议的详情工作台，定位到原始对话上下文。
* **飞书同步状态**：展示每个任务的飞书同步状态（未同步 / 已同步 / 同步失败）。


* **UI 设计要点**：
* 使用看板（Kanban）视图或列表视图，可切换。
* 优先级用颜色标签区分（urgent 红色、high 橙色、medium 黄色、low 灰色）。


* **后端对接**：`GET /meetings/{id}/actions`（遍历会议）、`PATCH /actions/{id}`、`DELETE /actions/{id}`、`POST /meetings/{id}/actions`、`POST /actions/{id}/sync/feishu`。

## 5. 决策追踪与冲突检测页 (Decision & Conflict)

**功能定位**：实现 PRD 亮点的"决策追踪"能力，帮助用户发现跨会议的决策变化与冲突。

* **核心功能**：
* **决策时间线**：按时间轴展示所有会议中提取的决策记录，标注来源会议和关键参与者。
* **冲突检测**：高亮显示与历史决策矛盾的新决策，标注冲突等级（low / medium / high）。
* **决策详情**：展开决策卡片可查看原文引用、关联会议、影响范围。
* **手动触发检测**：提供"检测冲突"按钮，调用后端冲突分析接口。


* **UI 设计要点**：
* 采用垂直时间线布局，冲突节点用红色警示标记。
* 冲突详情使用对比卡片（旧决策 vs 新决策）并排展示。


* **后端对接**：`GET /meetings/{id}/decisions`、`GET /meetings/{id}/conflicts`、`POST /meetings/{id}/conflicts/detect`。

## 6. 知识图谱页 (Knowledge Graph)

**功能定位**：可视化展示 PRD 亮点"图谱推理"能力，让用户直观理解会议中的人物、项目、任务、决策之间的关系。

* **核心功能**：
* **图谱可视化**：使用 ECharts 力导向图展示实体（人物、项目、任务、决策、风险）及其关系边。
* **交互探索**：点击节点展开关联实体，悬停显示详情卡片，支持缩放和拖拽。
* **自然语言查询**：输入问题后调用 Graph Agent，高亮图谱中的相关路径。
* **筛选与聚焦**：按实体类型筛选显示，支持按会议范围过滤。


* **UI 设计要点**：
* 全屏画布为主，左侧提供筛选面板。
* 不同实体类型使用不同颜色和图标（人物-蓝色圆形、项目-绿色方形、任务-橙色菱形、决策-紫色六边形、风险-红色三角形）。


* **后端对接**：`GET /meetings/{id}/graph`、`POST /graph/query`。

## 7. 跨会议记忆搜索页 (Memory Search)

**功能定位**：实现 PRD 亮点的"跨会议长期记忆"，让用户能从历史会议中检索相关信息。

* **核心功能**：
* **语义搜索**：输入自然语言查询，基于向量检索返回相关会议片段。
* **结果卡片**：展示匹配片段的原文摘要、来源会议、时间戳，支持一键跳转到对应会议详情。
* **上下文关联**：搜索结果附带 AI 生成的关联分析，帮助用户理解跨会议的信息联系。


* **UI 设计要点**：
* 搜索框居中突出，结果列表采用卡片式布局。
* 每张结果卡片包含来源会议标签和可点击的时间戳链接。


* **后端对接**：`POST /memory/search`。

## 8. 个人中心与集成设置 (Settings)

**功能定位**：管理第三方权限和系统参数。

* **核心功能**：
* **第三方集成**：绑定飞书（Feishu）账号，配置 Execution Agent 的同步开关。
* **模型配置**：选择偏好的 LLM 或 Whisper 模型等级（如果需要）。
* **偏好设置**：UI 颜色主题切换。


* **后端对接**：用户相关及第三方授权接口。

## 9. 登录 / 注册页 (Login) —— **最后实现**

**功能定位**：用户身份认证入口，MVP 阶段可跳过，所有接口暂不鉴权。

* **核心功能**：
* **登录表单**：邮箱 / 用户名 + 密码登录。
* **注册表单**：邮箱 + 密码 + 确认密码注册。
* **Token 管理**：登录成功后存储 JWT，Axios 拦截器自动附加 Authorization 头。
* **路由守卫**：未登录用户访问受保护页面时重定向到登录页。


* **UI 设计要点**：
* 左右分栏：左侧品牌展示区（Logo + 卡通猫 + Slogan），右侧表单区。
* 表单使用 Element Plus 的 Form 组件，支持实时校验。


* **后端对接**：`POST /auth/login`、`POST /auth/register`（需后端新增）。

## 10. 错误页面 (Error Pages)

**功能定位**：处理异常路由和服务器错误，提升用户体验。

* **核心功能**：
* **404 页面**：未匹配路由时展示，提供返回首页按钮。
* **500 页面**：服务异常时展示，提供重试按钮。


* **UI 设计要点**：
* 使用卡通猫的"困惑"或"睡觉"插画，保持与整体风格一致。
* 提供明确的操作引导（返回首页 / 重试）。


* **后端对接**：无。

## 🚀 后续开发建议

1. **优先实现 Dashboard 到 Workspace 的路由跳转**：确保 Pinia 里的 `meeting_id` 能够随路由正确更新。
2. **实现"空状态" (Empty States)**：在没有会议或 AI 还没分析完时，展示可爱风的占位图（如睡觉的猫咪），这能极大提升用户对 AI 延迟的容忍度。
3. **响应式设计**：考虑到用户可能在平板电脑上查看会议，建议使用 Tailwind CSS 做好基础的适配。
4. **统一布局组件**：建议先实现 AppLayout（侧边栏 + 顶栏 + 内容区），所有页面复用，避免重复布局代码。
5. **登录鉴权延后**：MVP 阶段所有 API 暂不鉴权，登录页和路由守卫放在最后实现，不影响核心功能开发。
