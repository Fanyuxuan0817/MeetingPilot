# MeetingPilot 开发手册：音文联动模块 (V1.0)

## 1. 核心原理：双向绑定逻辑

* **正向联动 (音 → 文)**：音频播放 $\rightarrow$ 触发 `timeupdate` $\rightarrow$ 更新 Pinia `currentTime` $\rightarrow$ 文本列表监听 `currentTime` $\rightarrow$ 自动高亮并滚动。
* **反向联动 (文 → 音)**：用户点击文本 $\rightarrow$ 获取文本 `start` 时间 $\rightarrow$ 调用 WaveSurfer `seekTo` $\rightarrow$ 音频跳转播放。

---

## 2. 状态中心：Pinia Store 配置

在 `src/stores/meeting.ts` 中，我们需要维护三个关键变量：

1. `currentTime`: 音频当前的播放秒数。
2. `transcripts`: 所有的转录片段数组。
3. `isPlaying`: 播放状态，用于同步 UI 按钮。

```typescript
export const useMeetingStore = defineStore('meeting', {
  state: () => ({
    currentTime: 0,
    isPlaying: false,
    transcripts: [] as TranscriptChunk[]
  }),
  actions: {
    updateTime(time: number) {
      this.currentTime = time;
    }
  }
})

```

## 3. 逻辑封装：`useWaveSurfer.ts`

这个 Composable 是音频控制的“大脑”。它负责初始化第三方库并向外暴露操作接口。

```typescript
import WaveSurfer from 'wavesurfer.js'
import { useMeetingStore } from '@/stores/meeting'

export function useWaveSurfer() {
  const store = useMeetingStore()
  let ws: WaveSurfer | null = null

  const initWaveform = (container: HTMLElement, url: string) => {
    ws = WaveSurfer.create({
      container,
      waveColor: '#E1BEE7', // 柔和紫色
      progressColor: '#9C27B0',
      height: 60,
      cursorColor: '#7B1FA2'
    })

    ws.load(url)

    // 核心：监听时间更新
    ws.on('timeupdate', (time) => {
      store.updateTime(time)
    })

    ws.on('play', () => store.isPlaying = true)
    ws.on('pause', () => store.isPlaying = false)
  }

  const seekTo = (seconds: number) => {
    if (!ws) return
    const duration = ws.getDuration()
    ws.seekTo(seconds / duration) // WaveSurfer 接收 0-1 的比例
  }

  return { initWaveform, seekTo, playPause: () => ws?.playPause() }
}

```

## 4. UI 实现：转录列表与高亮逻辑

在 `TranscriptList.vue` 中，我们需要通过 `currentTime` 计算出当前哪一段文本应该处于激活状态。

### 4.1 高亮判断逻辑 (Computed)

为了性能考虑，不要在模板里写复杂的逻辑，使用计算属性：

```typescript
// 寻找当前时间落在哪个 chunk 中
const activeId = computed(() => {
  const found = store.transcripts.find(
    chunk => store.currentTime >= chunk.start && store.currentTime <= chunk.end
  )
  return found?.id
})

```

### 4.2 自动滚动 (Watcher)

当 `activeId` 变化时，利用 `scrollIntoView` 自动定位。

```typescript
watch(activeId, (newId) => {
  if (newId) {
    const el = document.getElementById(`chunk-${newId}`)
    el?.scrollIntoView({ behavior: 'smooth', block: 'center' })
  }
})

```

## 5. 关键组件：`Waveform.vue`

这是你“可爱风” UI 的视觉中心。

```vue
<template>
  <div class="waveform-container bg-purple-50 p-4 rounded-xl shadow-sm">
    <div ref="waveformRef"></div>
    
    <div class="controls flex justify-center mt-2">
      <el-button @click="playPause" circle size="large">
        <el-icon><VideoPause v-if="store.isPlaying" /><VideoPlay v-else /></el-icon>
      </el-button>
    </div>
  </div>
</template>

<script setup lang="ts">
import { useWaveSurfer } from '@/composables/useWaveSurfer'
const props = defineProps<{ url: string }>()
const waveformRef = ref<HTMLElement | null>(null)
const { initWaveform, playPause } = useWaveSurfer()

onMounted(() => {
  if (waveformRef.value) initWaveform(waveformRef.value, props.url)
})
</script>

```

## 6. 鲁棒性考量 (Robustness Check)

为了防止系统因为音频加载失败或数据异常而崩溃：

1. **加载状态**：在 WaveSurfer `ready` 事件触发前，给波形图加一个 `v-loading` 遮罩。
2. **空数据处理**：如果后端返回的 `transcripts` 为空，列表应显示“暂无转录内容”，而不是报 `undefined` 错误。
3. **销毁逻辑**：在 `onUnmounted` 中调用 `ws.destroy()`，释放浏览器内存，防止页面切换导致的音频重叠。

---

### 💡 开发建议

既然你已经有了 `test_consistency.py`，在开始开发前，请务必运行它，确保 `TranscriptChunk` 的 `start` 和 `end` 字段在 TS 里不是 `string`（它们必须是 `number` 才能做 `currentTime` 比较）。
