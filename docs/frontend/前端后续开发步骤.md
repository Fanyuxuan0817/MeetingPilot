# 前端开发的具体实施步骤


#### 第一步：构建全局布局与导航（Dashboard）

* **任务**：实现会议列表页面，展示历史记录。
* **目的**：打通 Vue Router 路由，确保能从列表点击进入具体的“会议详情页”。
* **可爱风适配**：开始应用你喜欢的黄色系、圆角卡片布局。

#### 第二步：完善会议详情页（Workspace）

* **任务**：将你已经写好的**音文联动模块**集成到详情页中。
* **核心逻辑**：
* 对接 Pinia Store，将 Mock 的转录切片（Transcript Chunks）渲染成对话流。
* 实现“点击文字，音频跳转”以及“音频播放，文字高亮”的闭环。

#### 第三步：开发 AI 交互侧边栏（Agent Interaction）

* **任务**：设计右侧的 Agent 分析面板。
* **内容**：
* **摘要/待办看板**：渲染 `Summary Agent` 和 `Action Agent` 的静态 Mock 结果。
* **聊天窗口**：开发对话框 UI。
* **亮点功能**：实现“追溯定位”交互——点击 AI 回复中的时间戳，触发左侧音文模块跳转。

#### 第四步：文件处理与状态反馈

* **任务**：开发文件上传组件。
* **交互**：实现上传进度条、转录中的加载动画（Skeleton Screen）。因为真实转录很慢，好的前端反馈能极大提升用户体验。

---

### 3. 建议采取的策略：交替迭代法

虽然现在可以全力写前端，但为了项目最终落地，建议采取“纵向切片”的策略：

1. **写前端 UI**：把会议详情页画漂亮，接通 Mock 数据。
2. **接通后端主干**：回过头去把后端的“上传音频 -> Whisper 转录 -> 存入数据库”跑通。
3. **前后端联调**：将前端请求从 Mock 切换到真实 API。
4. **循环**：再开始写前端的 Agent 聊天窗口 -> 再去后端写 LangGraph 逻辑。

**总结建议：**
你现在可以**立刻开始构建前端的页面布局和各个 Agent 的展示组件**。只要你守住之前定义的 API 契约，即便后端 AI 部分还没写完，你的前端代码依然是有效且最终可直接使用的。
