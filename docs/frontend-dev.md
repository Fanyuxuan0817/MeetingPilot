# MeetingPilot 前端开发文档

## 技术栈

- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI 框架**: Element Plus / Ant Design Vue
- **状态管理**: Pinia
- **路由**: Vue Router
- **音频处理**: WaveSurfer.js
- **图表**: ECharts / D3.js (知识图谱)
- **HTTP 客户端**: Axios

---

## 页面模块

### 1. 会议管理模块

#### 1.1 会议列表页 `/meetings`

| 功能 | 说明 |
|------|------|
| 会议列表展示 | 卡片/列表视图切换 |
| 搜索过滤 | 按标题、日期、标签搜索 |
| 标签分类 | 侧边栏标签筛选 |
| 新建会议 | 弹窗/跳转创建页 |
| 导入音频 | 拖拽上传 + 文件选择 |

#### 1.2 创建会议页 `/meetings/create`

| 字段 | 类型 |
|------|------|
| 会议标题 | Input |
| 会议时间 | DateTimePicker |
| 参会人员 | TagInput |
| 会议标签 | TagInput |
| 音频上传 | Upload |
| 备注 | TextArea |

#### 1.3 会议详情页 `/meetings/:id`

| 区域 | 内容 |
|------|------|
| 头部信息 | 标题、时间、时长、参会人 |
| 音频播放器 | 波形 + 进度条 + 控制按钮 |
| 转录区域 | 分段文本 + 说话人 + 时间戳 |
| 纪要区域 | 结构化纪要展示 |
| 侧边栏 | 待办、决策、问答入口 |

### 2. 转录模块

```
功能:
- 分段展示转录文本
```

```
功能:

- 播放速度调节
- 音量控制
- 时间戳标记点
- 进度拖拽
```

## API 接口对接

### 会议相关

```
GET    /api/meetings          # 获取会议列表
POST   /api/meetings          # 创建会议
GET    /api/meetings/:id      # 获取会议详情
PUT    /api/meetings/:id      # 更新会议
DELETE /api/meetings/:id      # 删除会议
POST   /api/meetings/:id/upload  # 上传音频
```

### 转录相关

```
GET  /api/meetings/:id/transcript  # 获取转录内容
PUT  /api/meetings/:id/transcript  # 更新转录内容
```

### 纪要相关

```
GET  /api/meetings/:id/summary  # 获取纪要
PUT  /api/meetings/:id/summary  # 更新纪要
```

### 问答相关

```
POST /api/qa                    # 提交问题
GET  /api/qa/:meeting_id/history # 获取问答历史
```

### 待办相关

```
GET    /api/tasks           # 获取待办列表
PUT    /api/tasks/:id       # 更新待办状态
DELETE /api/tasks/:id       # 删除待办
```

### 决策相关

```
GET /api/decisions           # 获取决策列表
GET /api/decisions/conflicts # 获取冲突决策
```

### 知识图谱

```
GET /api/knowledge-graph     # 获取图谱数据
GET /api/knowledge-graph/search # 搜索节点
```

### WebSocket

```
WS /ws/transcript/:meeting_id  # 实时转录推送
WS /ws/qa/:meeting_id          # 实时问答
```

## 开发优先级

### 第一阶段 (MVP)

1. 用户登录/注册
2. 会议列表页
3. 创建会议 + 音频上传
4. 音频播放器
5. 转录展示
6. 纪要展示

### 第二阶段

1. 智能问答界面
2. 时间定位跳转
3. 待办列表页

### 第三阶段

1. 决策追踪
2. 飞书集成配置
3. 待办同步

### 第四阶段

1. 知识图谱可视化
2. 冲突检测提示
3. 高级设置

## 项目结构建议

```
src/
├── api/                 # API 接口
├── assets/              # 静态资源
├── components/          # 组件
│   ├── common/          # 通用组件
│   ├── layout/          # 布局组件
│   ├── meeting/         # 会议相关组件
│   ├── transcript/      # 转录相关组件
│   ├── qa/              # 问答相关组件
│   ├── task/            # 待办相关组件
│   └── graph/           # 图谱相关组件
├── composables/         # 组合式函数
├── router/              # 路由配置
├── stores/              # Pinia 状态管理
├── types/               # TypeScript 类型定义
├── utils/               # 工具函数
└── views/               # 页面视图
    ├── auth/
    ├── meetings/
    ├── tasks/
    ├── decisions/
    ├── graph/
    └── settings/
```
