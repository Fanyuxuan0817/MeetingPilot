# 1. 项目背景

在实际团队协作中，会议记录通常存在以下问题：

* 内容冗长，信息分散
* 决策信息难以回溯
* 待办事项容易遗漏
* 历史会议知识难沉淀
* 会后整理依赖人工

现有会议系统通常仅支持：

* 音频录制
* 自动转写
* 简单总结

缺少真正智能化的：

* 语义理解
* 历史记忆
* 自动追踪
* 决策分析

因此设计本系统，实现会议全流程智能协作。

# 3. 总体架构

## Base Platform + Multi-Agent Layer

双层架构。

---


# 5.2 Agent 说明

---

* 转录
* 分段
* 时间标记

---

## Agent 2：Summary Agent

负责：

* 摘要
* 纪要
* 决策提取

---

## Agent 3：Action Agent

负责：

* 待办提取
* 负责人识别

---

## Agent 4：Memory Agent

负责：

历史会议检索。

支持：

跨会话上下文。

---

## Agent 5：Graph Agent

负责：

图谱构建。

---

## Agent 6：Conflict Agent

负责：

决策比对。

---

## Agent 7：QA Agent

负责：

统一问答。

---

## Agent 8：Execution Agent

负责：

工具执行。

例如：

同步到：

* [飞书开放平台](https://open.feishu.cn/?utm_source=chatgpt.com)
